【File Storage 学习系列】文件存储的各种场景

发表于 2023-09-04 分类于 product read 阅读次数： 本文字数： 510 阅读时长 ≈ 1 分钟
  1. 企业应用上云
    • 作为部分企业应用上公共云的替代品
  2. 容器存储
    • 主要是用的是共享的能力，解决容器服务可能在短时间内读取 Docker File 拉起多个容器的场景
  3. 内容管理和 Web 服务
    • 作为文件系统所提供的一定程度上的基本能力，类似 HDFS
  4. AI 训练、基因计算、工业仿真等高性能计算
    • 高 IOPS，高带宽，低延迟要求
    • 海量小文件
  5. 媒体和娱乐工作流
    • 共享，以及扩容方便性，降低视频等处理成本

可以看出对文件存储的需求是有递进性的，最开始要求作为文件系统的能力。而后支持共享，以及作为云上的弹性的能力。然后支持高性能高带宽，海量小文件的极端需求场景。也是随着对于需求的变化而不断发展的过程。缓存类型的文件存储以及数据流动的产生，也是随着需求的演进而来的。

从历史到目前都是一个趋势：数据量越来越大，因此要求更低的成本以及更高的读写能力，对单机读写能力的不满足，所以要支持数据共享的能力，能够支持切片进行并行计算。

那么下一个需求是什么呢？又会提出什么更高的需求呢？

但需求越极端则需求数量越少，成本可能就越高。用一套文件存储解决所有的场景终究是不可能的，本也是一个平衡，怎么用最优的成本，解决更多的场景的需求。然后扩充产品线解决更多的场景，更好的支撑用户的需求场景。
